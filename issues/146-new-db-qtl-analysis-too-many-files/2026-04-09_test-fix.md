## 2. Integration Test Suite

`collect_test_data.sh` writes fixtures to `tests/integration_data/<profile>/`
(not `tests/integration_data/` directly). For the default `test` profile,
env vars must point at `tests/integration_data/test/...`, otherwise every
integration test skips with "TEST_DB_DIR not set or directory does not exist".

```bash
# Collect fixtures (runs the pipeline + copies db/ + assessment TSVs + GWA work files)
bash tests/collect_test_data.sh

# Run the integration suite against the collected fixtures
TEST_DB_DIR=tests/integration_data/test/db \
TEST_WORK_DIR=tests/integration_data/test/work \
TEST_LEGACY_ASSESSMENT=tests/integration_data/test/simulation_assessment_results.tsv \
TEST_DB_ASSESSMENT=tests/integration_data/test/db_simulation_assessment_results.tsv \
Rscript tests/run_tests.R
```

The test we added specifically for this fix —
`query_mapping_direct() matches query_for_threshold_analysis() schema and content`
at `tests/testthat/test-cross_validation.R:406` — requires `TEST_DB_DIR` to be
set to a populated DB directory. It will skip (not fail) if the path is wrong,
so confirm `ls tests/integration_data/test/db/mappings/` lists partitions
before trusting a "no failures" result.

## 3. Production-Scale Reproduction (intermediate-confidence signal)

Before re-running the full failing pipeline on Rockfish, reproduce one failing
`analyze_qtl.R` invocation against the real 16,608-file database and watch the
open-file count. If the new code path returns a populated dataframe while
holding only a handful of FDs, the fix is validated at production scale.

### Prerequisites

- SSH to the Rockfish login node (or `srun` onto a compute node with the
  `migrate-sims:20260202` container available).
- Path to the failing run's work directory (the one whose `.command.log`
  showed `IO Error: ... Too many open files in system`).
- Path to the checkout of this branch (`fix/db-qtl-io-error`) on Rockfish.

### Step 1 — Pick one failing task's work dir

```bash
# Path to the failing production run
FAILED_RUN=/scratch4/eande106/mckeowr1/Sims_ce-cb-ct_nqtl1-50_h2grid_50reps

# Find one ANALYZE_QTL work dir that crashed on "Too many open files"
grep -l "Too many open files" \
    "${FAILED_RUN}"/work/*/*/.command.log 2>/dev/null \
    | grep analyze_qtl \
    | head -1
# -> copy that path into WORK_DIR below
export WORK_DIR=/scratch4/eande106/mckeowr1/Sims_ce-cb-ct_nqtl1-50_h2grid_50reps/work/ab/cd1234...
```

### Step 2 — Recover the original CLI invocation

Each Nextflow task writes its exact command line to `.command.sh`. Pull the
`analyze_qtl.R` line out verbatim so the reproduction uses the real arguments
from the failing task:

```bash
cd "${WORK_DIR}"
cat .command.sh
# Copy the `analyze_qtl.R ...` line — keep every flag as-is.
```

### Step 3 — Point `R_SOURCE_DIR` at the patched branch

`analyze_qtl.R`'s `source()` calls read from `$R_SOURCE_DIR`. Override it to
the checked-out fix branch so the task runs against the patched
`R/queries.R` (with `query_mapping_direct()`) without rebuilding the container.

```bash
# Checkout of this repo on Rockfish, on branch fix/db-qtl-io-error
export FIX_CHECKOUT=/home/mckeowr1/code/nemascan-sims-nf
cd "${FIX_CHECKOUT}" && git status   # confirm branch = fix/db-qtl-io-error

# Override R_SOURCE_DIR for this shell
export R_SOURCE_DIR="${FIX_CHECKOUT}/R"

# Also put the patched analyze_qtl.R on PATH
export PATH="${FIX_CHECKOUT}/modules/db_migration/analyze_qtl/resources/usr/bin:${PATH}"
which analyze_qtl.R   # must resolve inside FIX_CHECKOUT
```

### Step 4 — Re-run the task inside its work dir

Run inside the same container image the pipeline uses so the R package set
matches production. Launch in the background so we can `lsof` its PID.

```bash
cd "${WORK_DIR}"

# Same container the process config uses (see conf/rockfish.config)
IMG=/scratch4/eande106/singularity/migrate-sims_20260202.sif

# Rebuild the CLI from .command.sh. Example shape — replace with the real flags:
singularity exec \
    --env R_SOURCE_DIR="${R_SOURCE_DIR}" \
    --bind "${FIX_CHECKOUT}":"${FIX_CHECKOUT}" \
    --bind "${FAILED_RUN}":"${FAILED_RUN}" \
    "${IMG}" \
    analyze_qtl.R \
        --group ce.full --maf 0.05 --nqtl 5 --effect gamma \
        --rep 1 --h2 0.5 --mode loco --type nopca \
        --threshold BF --cv_maf_effective 0.05 --cv_ld 0.8 \
        --base_dir "${FAILED_RUN}/db" \
        --ci_size 150 --snp_grouping 1000 --alpha 0.05 \
    > analyze_qtl.stdout 2> analyze_qtl.stderr &

RPID=$!
echo "analyze_qtl.R PID=${RPID}"
```

### Step 5 — Watch the FD count while it runs

Sample `lsof` a few times. The old code path opens ~16,608 parquet files at
`CREATE VIEW` time; the new path should never exceed a handful.

```bash
# Peak open-file count for that PID (polled once per second until exit)
while kill -0 "${RPID}" 2>/dev/null; do
    n=$(lsof -p "${RPID}" 2>/dev/null | wc -l)
    printf '%s fds=%s\n' "$(date +%T)" "${n}"
    sleep 1
done | tee analyze_qtl.lsof.log

wait "${RPID}"
echo "exit=$?"
```

Pass criteria:

- Exit status `0`.
- `analyze_qtl.lsof.log` never shows more than ~20–30 open FDs attributable
  to the R process (stdlib + libs + a couple of parquet files). Definitely
  not thousands.
- `*_qtl_regions.tsv` exists in `${WORK_DIR}` and is non-empty (`wc -l` > 1
  header row, or at minimum a valid zero-QTL file matching the legacy shape).
- `analyze_qtl.stderr` shows no `Too many open files` and no
  `Mapping partition not found` errors.

### Step 6 — Clean up

```bash
# The reproduction writes into the original work dir; remove the artifacts
# so a future `-resume` doesn't pick them up in an unexpected state.
rm -f "${WORK_DIR}/analyze_qtl.stdout" \
      "${WORK_DIR}/analyze_qtl.stderr" \
      "${WORK_DIR}/analyze_qtl.lsof.log"
# Leave the original .command.* files and the newly written *_qtl_regions.tsv
# in place — the full `-resume` run will overwrite them.
```

If all three signals (exit 0, low FD count, populated regions file) hold on a
real failing task, proceed to the full `-resume` run in section D of
`fix.md`.

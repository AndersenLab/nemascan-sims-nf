## Solution

### Approach

All 5 errors are test-only: no production code required changes. The
fixes align the failing test call sites with the APIs as they currently
exist.

- **Issue Group 1:** Replace each `generate_mapping_id(params)` call
  with the correct 3-step resolution chain. Since the parsed-filename
  params list lacks `species`, `vcf_release_id`, and `ms_ld` (needed by
  `generate_marker_set_id()`), the test must call
  `read_marker_set_metadata()` first to recover those values from
  `TEST_DB_DIR`. Inside for-loops, a `NULL` result uses `next` rather
  than `skip()` so iteration continues.
- **Issue Group 2:** Replace each 3-arg `read_marker_set()` call with
  the correct 5-arg form, prepended by a `read_marker_set_metadata()`
  lookup and a `NULL` skip guard. This matches the pattern already
  present in `test-db_structure.R:40–45`.
- **Documentation:** Added a `@note` to the `generate_mapping_id()`
  roxygen block in `R/database.R` documenting the required resolution
  chain for callers holding a filename-parsed params object.

``` r
#'   params list. To resolve a mapping ID from a filename-parsed params object, first call
#'   read_marker_set_metadata() -> generate_marker_set_id() -> generate_trait_id() ->
#'   generate_mapping_id(trait$hash, params$algorithm, params$pca)$hash.
```

### Rationale

Updating the tests rather than adding convenience wrappers is correct
because:

1.  The params list returned by `parse_inline_gwa_filename()` is
    structurally incomplete for a `generate_mapping_id` wrapper — it
    lacks `species`, `vcf_release_id`, and `ms_ld`, so any wrapper would
    need those as additional arguments, negating the simplification.
2.  `read_marker_set_metadata()` recovers the missing fields cleanly;
    the pattern is already established at `test-db_structure.R:40–45` in
    the same file as one of the failing tests.
3.  A `read_marker_set()` overload with optional
    species/vcf_release_id/ms_ld would silently pick the first-matching
    metadata row under pop+maf collisions — a latent correctness risk
    that the explicit 5-param API was designed to prevent (per
    CLAUDE.md).
4.  Zero production files changed; regression surface is contained to
    two test files.
